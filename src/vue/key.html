key

  • 1. key
  • 2. 列表渲染
  • 3. 强制替换

1. key

支持字符串string或数值number类型的值

主要用在 Vue 的虚拟 DOM 算法，在新旧 nodes 对比时辨识 VNodes。

不使用 key，Vue 会使用一种最大限度减少动态元素并且尽可能的尝试就地修改/复用相同类型元素的算法。

使用 key 时，会基于 key 的变化重新排列元素顺序，并且会移除 key 不存在的元素。

2. 列表渲染

当 Vue 正在更新使用 v-for 渲染的元素列表时，它默认使用“就地更新”的策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新修改每个元素，并且确保它们在每个索引位置正确渲染

这个默认的模式是高效的，但是只适用于不依赖子组件状态或临时 DOM 状态 (例如：表单输入值) 的列表渲染输出

  • 不使用key

    <template>
  <div id="app">
    <div>
      <input type="text" v-model="name">
      <button @click="add">添加</button>
    </div>
    <ul>
      <li v-for="(item, index) in list">
        <input type="checkbox">{ {item.name}}
      </li>
    </ul>
  </div>
</template>
<script>
export default {
  name: "App",
  data() {
    return {
      name: "",
      newId: 3,
      list: [
        { id: 1, name: "aaaa" },
        { id: 2, name: "bbbb" },
        { id: 3, name: "cccc" }
      ]
    }
  },
  methods: {
    add() {
      this.list.unshift({ id: ++this.newId, name: this.name })
      this.name = ""
    }
  }
}
</script>

选中一个值

添加一个值，checkbox选中状态非预期，三个checkbox处于临时状态，就地复用；aaaa修改为dddd，bbbb修改为aaaa，cccc修改为bbbb，新增一个li，值为cccc。就地复用时checkbox和name值处于分离状态

  • 使用key

    <ul>
    <li  v-for="(item, index) in list" :key="item.id">
        <input type="checkbox">{ {item.name}}
    </li>
</ul>

    新旧node根据key对比时，checkbox和name值处在一种绑定状态，key不相同添加，key相同移动顺序

  • 使用索引作为key

    <ul>
    <li  v-for="(item, index) in list" :key="index">
        <input type="checkbox">{ {item.name}}
    </li>
</ul>

    数组索引在增删时会动态修改，[0,1,2]变为[0,1,2,3]，更新过程和不使用key时类似，key相同的修改text，不相同的新建

3. 强制替换

  • 完整地触发组件的生命周期钩子

  • 触发过渡

    <transition>
  <span :key="text">{ { text }}</span>
</transition>

    当 text 发生改变时，<span> 总是会被替换而不是被修改，因此会触发过渡
