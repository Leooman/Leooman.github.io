变更检测

change detector变化检测器

  • 触发时机：基本上所有的异步事件发生（并且回调函数已经执行完毕）以后，都需要触发change detection（因为此时进程的状态可能已经发生改变），如Events ，XHR ，Timers
  • 单向数据流：沿着组件树进行变化检测，检查完父组件以后再检查子组件，在检查父组件的时候可能会更新子组件中的绑定，但是在检查子组件的时候（此时父组件已经检查完毕）不会更新父组件的数据，只需要执行一次Change Detection，就能使得view与组件中的数据一致。在AngularJS中，由于在检查一个组件的时候可能会改动另一个组件中的数据，因此需要多次检查，直到数据“稳定”下来
  • 专用change detector：每个组件都有一个自己的change detector(Angular compiler为每个component编译生成专门检测它的view的代码)，这使得每个change detector的检测代码非常地简单高效

多态(polymorphic)：通用性往往意味着多态的存在。多态的含义是，一段代码，多次执行，但是每次执行所操作的对象都不是同一个对象，甚至这些对象的shape相差巨大，这种代码难以被优化。大量使用单态（monomorphic）是Angular速度爆炸的重要原因！除了Change Detection方面，Angular在其他地方也使用了单态的优化思想

ChangeDetectionStrategy 变更检测策略

  • OnPush = 0：若输入属性没有发生变化，组件的变化检测将会被跳过
  • Default = 1：组件的变更检测一直执行，默认采用此策略，因为即使@Input()对象的引用没有变，其中的属性可能已经发生变化（JavaScript对象的动态性），变化检测需要将这种变化也反映在view和子组件上

ChangeDetectorStatus 变更检测状态

  • CheckOnce = 0：检测变化后变更检测器模式切换为Checked
  • Checked = 1：变更检测会被跳过
  • CheckAlways = 2：检测变化后变更检测器模式保持CheckAlways
  • Detached = 3：已将组件从变更检测树中分离
  • Errored = 4：绑定检查报错或调用生命周期方法与当前状态不一致，变更检测停止工作
  • Destroyed = 5：变更检测器销毁

ChangeDetectorRef 变更检测机制

export declare abstract class ChangeDetectorRef {
  markForCheck(): void;
  detach(): void;
  detectChanges(): void;
  checkNoChanges(): void;
  reattach(): void;
}
  • markForCheck(): void：标记为脏状态进入变更检测周期的检测
//=> OnPush策略下的组件会进入checked状态，想要进行变更检测要调用markForCheck()
@Component({
  selector: 'my-app',
  template: `Number of ticks: {{numberOfTicks}}`
  changeDetection: ChangeDetectionStrategy.OnPush,
})
class AppComponent {
  numberOfTicks = 0;
  constructor(private ref: ChangeDetectorRef) {
    setInterval(() => {
      this.numberOfTicks++;
      this.ref.markForCheck();
    }, 1000);
  }
}
  • detach(): void：将视图从变更检测树中分离不进行检测
  • detectChanges(): void：该组件到各个子组件执行一次变化检测，可用于自主控制视图更新频率 
//=> 将组件从变更检测树中分离出去，实时变化的数据改变为每隔5s触发一次变更检测
//=> data.service.ts
export class DataService {
  private arr:number[];
  get data() {
      let array:numner[];
      array=this.arr=this.arr ? this.arr.map(x=>x+=5) : [1,2,3,4,5];
      return array;
  }
}

//=> giantList.component.ts
@Component({
  selector: 'giant-list',
  template: `
    <li *ngFor="let d of dataProvider.data">Data {{d}}</li>
  `,
})
export class GiantListComponent {
  constructor(private ref: ChangeDetectorRef, private dataProvider: DataService) {
    ref.detach();
    setInterval(() => {
      this.ref.detectChanges();
    }, 5000);
  }
}

@Component({

  `,

class App {
}
  • reattach(): void：将视图重新附加到变更检测树中进入检测
//=> 切换视图在变更检测树中的分离或附加状态
//=> data.service.ts
export class DataService {
  data = 1;
  constructor() {
    setInterval(() => {
      this.data = this.data * 2;
    }, 500);
  }
}

//=> liveData.component.ts
@Component({
  selector: 'live-data',
  inputs: ['live'],
  template: 'Data: '
})
export class LiveDataComponent {
  constructor(private ref: ChangeDetectorRef, private dataProvider: DataService) {}
  set live(value:boolean) {
    value ? this.ref.reattach() : this.ref.detach()
  }
}

//=> app.component.ts
@Component({
  selector: 'my-app',
  providers: [DataProvider],
  template: `
    Live Update: <input type="checkbox" [(ngModel)]="live">
    <live-data [live]="live"><live-data>
  `,
})
export class AppComponent {
  live = true;
}
  • checkNoChanges(): void：检测组件及其子组件，存在变化则报错。用于开发模式确保变化检测期间没有引进其他变化
