组件

description

组件是Angular应用程序中UI的最基本构建块。

component是对directive的扩展，不同于directive的是，component始终具有模板template并且每个模板中的元素只能实例化一个组件。

一个组件必须属于NgModule模块，在NgModule的declarations字段中声明，以便其他的组件或应用使用。

how to use

import { Component } from '@angular/core';
@Component({
    changeDetection?: ChangeDetectionStrategy,     //=> 定义使用的变更检测策略
    viewProviders?: Provider[],                    //=> 设置组件或子组件可以使用的服务
    moduleId?: string,                             //=> 包含该组件的模块的模块ID，用于解析模板和样式的相对路径
    templateUrl?: string,                          //=> 外部模板文件地址
    template?: string,                             //=> 内联模板
    styleUrls?: string[],                          //=> 外部样式文件地址
    styles?: string[],                             //=> 内联样式
    animations?: AnimationEntryMetadata[],         //=> 定义动画
    encapsulation?: ViewEncapsulation,             //=> 指定模板和样式封装规则
    interpolation?: [string, string],              //=> 重写默认的插值运算符（默认为'{{'和'}}'）
    entryComponents?: Array<Type<any> | any[]>,    //=> 组件定义时需要预编译的组件
    preserveWhitespaces?: boolean                  //=> 是否保留编译后的空白字符
})
export class MyComponent{
}

changeDetection

组件实例化时创建一个变化检测器，用来检测组件的绑定，changeDetection属性定义了变化检测方式

@Component({
    selector:'cmp',
    changeDetection:ChangeDetectionStrategy.OnPush || 0,
    template:`Number of ticks: {{numberOfTicks}}`
})

export class CmpComponent{
    numberOfTicks = 0
    constructor() {
        setInterval(() => {
            this.numberOfTicks ++
        },1000)
    }
}

viewProviders

//=> role.directive.ts
@Directive({
    selector:'greet',
    exportAs:'G'
})
export class GreetDirective{
    greet:GreetService;
    constructor(greeter:GreetService){
        this.greeter = greeter;
    }
}
//=> greet.service.ts
import { Injectable } from '@angular/core'
@Injectable()
export class GreetService{
    constructor(){}
    greet(name:string){
        return "hello"+name;
    }
}
//=> app.component.ts
@Component({
    selector:'my-app',
    templateUrl:'./app.component.html',
    viewProviders:[
        GreetService
    ]
})
//=> app.component.html
<greet #g='G'>{{g.greeter.greet('Angular')}}</greet>

moduleId

解决模板与样式文件的相对路径问题，可以设置为module.id 

@Directive({
  selector: 'someDir',
  moduleId: module.id
})
class SomeDir {
}

templateUrl

指定组件的模板文件地址，每个组件只可以定义templateUrl或template中的一个，之后定义的会覆盖之前定义的

@Component({
    selector:'my-app',
    templateUrl:'./app.component.html',      //=> 视图不渲染
    template:`<p>Angular</p>`                //=> 视图渲染
})

template

指定组件的内联模板，每个组件只可以定义templateUrl或template中的一个，之后定义的会覆盖之前定义的

styleUrls

指定样式文件地址的列表集合

@Component({
    selector:'my-app',
    templateUrl:'./app.component.html',
    styleUrls:['./app.component.css','./css/demo.css']
})

styles

指定组件的内联样式

@Component({
    selector:'my-app',
    templateUrl:'./app.component.html',
    styles:[`p{background:#f00;color:#fff;}`]
})

encapsulation

定义了组件中的模板和样式该如何封装：

  • ViewEncapsulation.Emulated ：仿真模式，无 Shadow DOM， 通过Angular来模拟类似Shadow DOM的行为 ，使样式不受外部影响
@Component({
  template: `<h1>test</h1>`,
  styles: [`h1 { color: #f50; }`],
  encapsulation: ViewEncapsulation.Emulated || 0
})
//=>  <style>h1[_ngcontent-c0] { color: #f50; }</style>
    <h1 _ngcontent-c0>test</h1>
  • ViewEncapsulation.Native ：使用shadow roots影子根（如果浏览器支持Shadow-DOM）
@Component({
  template: `<h1>test</h1>`,
  styles: [`h1 { color: #f50; }`],
  encapsulation: ViewEncapsulation.Native || 1
})
//=>  #shadow-root (open)
        <style>h1 { color: #f50; }</style>
        <h1>test</h1>
  • ViewEncapsulation.None ：无任何封装行为，作为全局样式
@Component({
  template: `<h1>test</h1>`,
  styles: [`h1 { color: #f50; }`],
  encapsulation: ViewEncapsulation.None || 2
})
//=>  <style>h1 { color: #f50; }</style>
      <h1>test</h1>

当encapsulation在组件中没有被定义时使用CompilerOptions里定义的默认值ViewEncapsulation.Emulated,并提供一个CompilerOptions来重写此值

如果encapsulation设置为ViewEncapsulation.Emulated并且组件中没有定义styles或者styleUrls,encapsulation会自动切换为ViewEncapsulation.None

interpolation

@Component({
    selector:'my-app',
    template:`<p>Number of ticks: $$ numberOfTicks $</p>`,
    interpolation:["$$","$"]
})

entryComponents

指定组件定义时需要编译的组件列表，对于列表中的每个组件，Angular会创建一个ComponentFactory并且保存在ComponentFactoryResolver中

preserveWhitespaces

当preserveWhitespaces设置为false时，多余的空白符（正则匹配\s）会从已编译的模板中移除，很大程度上减小AOT（Ahead-of-Time）预编译模式生成的代码大小且加速视图创建*

模板转换处理会潜在的影响页面布局，为了最小化这种影响，从Angular6开始preserveWhitespaces默认设置为false。可以修改AOT编译器的preserveWhitespaces选项使应用内的所有组件都保留空白

当前执行遵循以下规则：

  • 模板开始和结束的空白都会被删除
  • 仅由空白组成的文本节点会被删除：<button>Action 1</button> <button>Action 2</button>被处理为<button>Action 1</button><button>Action 2</button>
  • 文本节点中一系列的空白会被替换为一个空格：<span>\n some text\n</span>被处理为<span> some text </span>
  • 如果空白是必不可少的，可以用<pre>或<textarea>标签包裹保留空白

尽管有默认的preserveWhitespaces:false处理空白，依然可以在DOM元素上添加ngPreserveWhitespaces属性保留空白

<div ngPreserveWhitespaces>
    whitespaces are preserved here
    <span>    and here </span>
</div>

或者使用&ngsp;作为文本节点，Angular编译器会转换为空格（多个&ngsp;会被编译为一个）：

<a>Spaces</a>&ngsp;<a>between</a>&ngsp;&ngsp;&ngsp;<a>links.</a>
//=> 编译为 <a>Spaces</a> <a>between</a> <a>links.</a>

animations

通过监听元素上的状态改变来执行animations动画，动画的状态变化存放在trigger中，state声明状态，transition控制状态变化

@Component({
    selector: 'animation-cmp',
    templateUrl: 'animation-cmp.html',
    animations: [
      trigger('myTriggerName', [
        state('on', style({ opacity: 1 }),
        state('off', style({ opacity: 0 }),
        transition('on => off', [
          animate("1s")
        ])
      ])
    ]
 })
 <div @myTriggerName="expression">...</div>
 //=> expression为当前动画执行到的状态值
